Tooling

Documentation

Document artifacts are prolific in every software development life cycle methodology. In general, a project will have at least three variations on project documentation. Each of the document editions are aligned for different resource skillsets. The major three are executive, managment and technical editions of the project documentation. While the executive edition may provide an overview of a project including a timeline and finaicial analysis it lacks the technical grit. Management editions will most often include resource managment requirements including finaincials at a departmental level. The technial editions will provide the details required to undertake the process of development and lack the finaical and recource management details.

The most common word processors found in project managment:

  • Microsoft Word
  • OpenOffice
  • LibreOffice

Project Managment

Project management tools come in many feature-set variations. Project managment tools are used to create the work breakdown structure (WBS), allocate and track resources (developers), predict milestone and project completion dates in addition to other project details.

The most common tool used for project management is Microsoft Project. The tool has been equiped in resent years with remote team features and the latest version includes a cloud based converged feature set. Convergens includes Porject Online and Team Foundation Server (TFS) integration.

Unified Modeling Language

Unified Modeling Language (UML) is a generalized modeling language used in software development. Unified Modeling Language artifacts can be divided into two main types, structural and behavioral diagrams. There are currently fourteen different ULM diagrams. Some UML artifacts include:

  • Class Diagram
  • Component Diagram
  • Deployment Diagrams
  • Use Case Diagram
  • Activity Diagram
  • Sequence Diagram

The popularity and useful nature of UML has generated many software implementations of tooling. There is just about any flavor of UML tool. Some are SaaS based tools others are traditional desktop applications. Here are a few of the most popular:

  • Microsoft Viso
  • Visual Studio
  • Unbrello CST338 UML Sample
UML Support was removed from Microsoft Viso after the last Enterprise Archtecht edition and the feature-set was moved to Visual Studio Ultimate Edition.   • Team Foundation Server
  • Microsoft Project Online
